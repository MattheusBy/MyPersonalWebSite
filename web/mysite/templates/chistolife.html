Project description

The main aim of this project is to create a completely finished project for the company, including all the necessary functionality for cleaning of upholstered furniture. My mate had a furniture dry cleaner company and he acted as a customer. We discussed exactly what functions of the site he'd like to see for himself, which I have subsequently implemented. The main task was to create a website, which visiting the client can independently obtain the maximum of information for him without contacting managers.

I decided to use the Flask framework because it allows to implement a wide range of options, at the same time, this framework isn't overloaded with tools out of the box. Connecting the database and the user registration system, I considered irrelevant for this application. I have made a decision to implement the following site structure:

  • The index page contains a banner with the name of the company, examples of work before and after cleaning, a section about the benefits of the company;
  • The page with current prices for each type of furniture and the prices for additional services;
  • The page describing the process and steps of cleaning;
  • The page with an online calendar with a schedule of cleanings by date and time. It needs for the reason that the client could find out and choose in advance convenient date and time for him the date and time of the submitted. The calendar is filling in by the manager when the client have confirmed the order.
  • The page containing contact information and links to social networks
  • The page allows the client to independently fill out the form with the necessary information and the obligatory attachment of a photo of furniture for an accurate assessment of the cost of services, since the final cost of cleaning doesn't always correspond to the price in the price list and depends on the degree of contamination, design features of the product, etc. This option allows the client doesn't waste time choosing a convenient messenger for him to contact the manager, and send information directly from the site. The message comes via Telegram to the manager and after evaluation the customer receives the final price for the cleaning of his product via SMS.
  • Implemented custom design for 404 and 500 error pages.

The additional task for me was to create an adaptive and pleasant interface. To solve this problem, I used Bootstrap 5, Canva, bootsnipp.com, svgwave.in, icons8.com.

The Pytest framework, Coverage tool, SeleniumWebDriver were used to test the application. Application coverage with tests is 92%.

The project uses the capabilities of Docker and Docker Compose to simplify the process of project implementation on the server.
